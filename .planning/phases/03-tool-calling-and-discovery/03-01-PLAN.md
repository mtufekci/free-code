<objective>
Add tool call request formatting to translate Anthropic tool schemas into Ollama's `tools` array format.
</objective>

<purpose>
When the system sends tool calls to Ollama, the request must use Ollama's expected format: `{type: "function", function: {name, description, parameters}}`. Currently `translateRequestToOllama()` ignores the `tools` parameter entirely.
</purpose>

  <action>
Update `translateRequestToOllama()` (around line 133) to handle the `tools` parameter:

1. After building `ollamaMessages` and before creating the return object, check if `params.tools` exists and is non-empty
2. If tools exist, translate each Anthropic tool to Ollama format:
   - Anthropic: `{name: string, description?: string, input: Record<string, unknown>}`
   - Ollama: `{type: "function", function: {name, description, parameters}}`
3. Add the `tools` array to the returned `OllamaChatRequest`
4. Ensure `options.num_ctx` is NOT set here (that belongs to MODL-03)

The Ollama format (verified from api.md):
```typescript
{
  "type": "function",
  "function": {
    "name": "get_weather",
    "description": "Get weather for a city",
    "parameters": { "type": "object", "properties": {...}, "required": [...] }
  }
}
```

Note: Anthropic's `input` is already JSON Schema format, so pass it through directly as `parameters`.
