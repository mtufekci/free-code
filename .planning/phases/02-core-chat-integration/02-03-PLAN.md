# PLAN-03: Integrate Ollama Client into API Routing

**Phase:** 2 - Core Chat Integration  
**Created:** 2026-04-01  
**Requirements:** CHAT-01, CHAT-02, CHAT-03, CHAT-04

## Context

The existing `getAnthropicClient()` in `src/services/api/client.ts` creates clients for all providers (Anthropic, Bedrock, Vertex, Foundry). We need to add Ollama routing so that when `CLAUDE_CODE_USE_OLLAMA=1`, the Ollama client is returned instead of the Anthropic client.

**Key constraint from research:**
> The `queryModel()` generator is provider-agnostic at the stream level. The Ollama client must provide the same interface as the Anthropic client.

## Architecture

```
getAPIProvider() → 'ollama' (already returns 'ollama' when enabled)
       ↓
getAnthropicClient() → detects 'ollama' provider → returns createOllamaClient()
       ↓
queryModel() in claude.ts → uses client.beta.messages.create() → WORKS with Ollama client
```

## Implementation

### Modify: `src/services/api/client.ts`

**Option A (Recommended):** Add Ollama check in `getAnthropicClient()`

```typescript
// Add near the top of getAnthropicClient(), after the Bedrock/Vertex/Foundry checks:

if (getAPIProvider() === 'ollama') {
  const { createOllamaClient } = await import('./ollama.js')
  return createOllamaClient() as unknown as Anthropic
}
```

**Option B:** Create separate `getClient()` function that routes to different clients based on provider. This would require more changes to `claude.ts`.

**Option A is preferred** because:
1. Minimal changes to existing code
2. The Ollama client mimics the Anthropic interface
3. `claude.ts` doesn't need modification

### Modify: `src/services/api/claude.ts` (if needed)

Check if any provider-specific logic in `queryModel()` needs Ollama handling:

1. **TTFT calculation** - Works with any stream that yields events
2. **Usage tracking** - Ollama doesn't provide token counts (may need to handle undefined)
3. **Error handling** - Should already work via the client's error responses
4. **Timeout handling** - Uses AbortSignal which the Ollama client supports

**Likely no changes needed** to `claude.ts` if the Ollama client interface is correct.

### Modify: `src/utils/model/ollama.ts`

Add helper function for normalizing Ollama model names if needed:

```typescript
/**
```
